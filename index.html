GU2D

Introduction

GU2D is a library to cretate 2D videogames for the PlayStation 2 exclusive on PlayLua (IDE), providing C/C ++ to programming. It is based on primitives extracted from open source projects, therefore I do not guarantee that it is free of Bugs.
It is important that you analyze the examples (PlayLua>Help>Examples>GU2D) so that you can understand how to use the library to program your game.

Functionalities

  • Textures: BMP,JPG,PNG,TGA
  • Audio files: XM, WAV
  • Graphic primitives: points, lines and quads
  • World system
  • 2D animation functions
  • Physics
  • Math functions
  • Pad functions (only one)

Game assets

During the compilation of a program (main file), PlayLua (GU2D MODE) compiles into a file (Game Assets), all the textures located in the folder called "textures", which must be in the same folder as the main file. This means that loading of a texture is done from memory and not from disk. Additionally within this folder, the textures must not be in subfolders.

Audio files

Audio files do not yet fall into the Game Assets category, therefore these files are loaded from disk.

Structure of a program

#include <gu2d.h>
// Here includes, variables, functions, ...
int main(int argc, char * argv[]){
guInit();
// Here loading textures, sounds, music ...
while(1){
guBeginScene();
// Here loop of scene ...
guEndScene();
}
return(1);
}

Drawing functions

The functions that allow drawing are: guText() guPoint(), guLine(), guRect(), guQuad(), guImage(), guRepeatQuad() and guPlayAnimator(). Additionally, all these functions except guText(), can be contextualized in a world where a region called "camera", whose dimensions are equal to the dimensions of the screen, is what is shown on the screen. Such contextualization can be done through functions such as guEnableWorld() to enable this contextualization, guWorld() to establish the dimensions of the world and guCamera() to establish the position of the "camera". 2D Animation

To animate a character you must have two things: a sprite sheet, and a Animator (variable of type GUAnimator) dedicated to the character. If the character has animations on the left and right side, the sprites for such animations should be on the sprite sheet. With the Animator you can, among iother things, add poses (sprites), add animations, set the animation side, run the animation and draw it.

Physics

Physics has the following characteristics: colliders are boxes (rectangles), there is no rotation, and gravity is allowed. The way to use this physics is explained in the following code segment:

#include <gu2d.h>
// Here includes, variables, functions, ...
int main(int argc, char * argv[]){
guInit();
// Add boxes to the system
while(1){
guBeginScene();
// Establish velocities to the boxes that require it.
guPlayPhysics();
// Drawings acoording to physics data.
guEndScene();
}
return(1);
}
